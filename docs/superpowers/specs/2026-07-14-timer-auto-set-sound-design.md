# AZIM.FIT — автозачёт подхода и звук таймера

## Цель

Убрать необходимость вручную отмечать каждый подход перед запуском отдыха. Нажатие кнопки отдыха упражнения должно одной операцией отметить следующий незавершённый подход и запустить глобальный таймер. При окончании таймера приложение должно один раз воспроизвести заметный звуковой паттерн длительностью не менее пяти секунд.

## Поведение

- Кнопка `90 сек` и другие кнопки отдыха сохраняют текущий внешний вид.
- Каждое нажатие отмечает ровно один следующий незавершённый подход выбранного упражнения.
- Количество выполненных подходов не может превысить запланированное.
- Если все подходы уже отмечены, таймер всё равно запускается, но результат упражнения не меняется.
- Повторное нажатие во время активного таймера начинает новый отсчёт и отмечает следующий подход, если он существует.
- Ошибочно отмеченный подход по-прежнему можно снять вручную существующими кнопками подходов.
- Для будущих, завершённых и пропущенных тренировок автозачёт недоступен так же, как текущий ручной учёт.

## Архитектура и поток данных

Store получает новое атомарное действие `WORKOUT_START_REST`. Reducer проверяет тренировку и упражнение, увеличивает `completedSets` максимум до `sets`, а затем создаёт `activeTimer` для того же упражнения. Подход и таймер сохраняются вместе в одном новом состоянии, поэтому промежуточного состояния между двумя dispatch не возникает.

`WorkoutCard` продолжает вызывать один обработчик кнопки отдыха. `App` подготавливает аудиоконтекст в рамках этого пользовательского нажатия и отправляет одно действие в reducer. Схема `AppStateV2` и миграция данных не меняются.

## Звуковой сигнал

- Сигнал генерируется локально через Web Audio API и не требует аудиофайла или сети.
- Аудиоконтекст создаётся или возобновляется только после нажатия пользователем на кнопку таймера, чтобы соответствовать ограничениям мобильных браузеров.
- При переходе активного таймера из положительного остатка в `expired` проигрываются пять двухтональных импульсов с интервалом 1,2 секунды. Последний тон завершается примерно через 5,15 секунды после начала сигнала.
- Один и тот же таймер не может прозвучать повторно при следующих рендерах.
- При загрузке уже истёкшего таймера звук не воспроизводится, чтобы приложение не начинало звучать неожиданно при открытии.
- Если браузер не поддерживает Web Audio или блокирует звук, таймер и автозачёт продолжают работать без ошибки.

## Ошибки и граничные случаи

- Отсутствующая тренировка, упражнение, нулевой отдых или недоступный для учёта статус не меняют количество подходов.
- Неподдерживаемый звук деградирует бесшумно.
- Пауза, продолжение, добавление 30 секунд и отмена таймера не отмечают дополнительные подходы.
- Звук проигрывается только при естественном достижении `00:00`, а не при отмене таймера.

## Проверки

- Unit: атомарное действие отмечает следующий подход и запускает таймер.
- Unit: повторные нажатия последовательно отмечают подходы и не превышают `sets`.
- Unit: полностью выполненное упражнение запускает таймер без изменения результата.
- Unit: недоступные статусы и некорректные идентификаторы не меняют тренировку.
- Component: нажатие кнопки отдыха вызывает единый обработчик.
- Component/unit: звуковой паттерн создаёт десять тонов, длится не менее пяти секунд, запускается один раз при переходе в `expired`, не проигрывается после загрузки уже истёкшего таймера и не ломает приложение без Web Audio.
- Regression: ручная отметка, пауза, продолжение, `+30 секунд`, отмена и восстановление таймера после reload остаются рабочими.

## Не входит в изменение

- Выбор мелодии и громкости.
- Отдельная настройка отключения звука.
- Фоновый звук после полного завершения PWA операционной системой.
- Изменение визуального дизайна карточки упражнения или таймера.
