# KEEP AT IT — бренд, account header и аватары

## Цель

Завершить переход от AZIM.FIT к универсальному бренду KEEP AT IT и сделать состояние аккаунта понятным прямо в основном интерфейсе. Гость должен сразу видеть возможность входа, а авторизованный пользователь — свой аватар и состояние синхронизации, не перегружая мобильный хедер.

## Бренд

- Отображаемое имя продукта: `KEEP AT IT`.
- Основной домен: `keepatit.me`.
- Существующий лаймовый знак-молния, палитра и типографика сохраняются; это обеспечивает преемственность PWA и не требует нового логотипа в рамках этой задачи.
- Все видимые названия AZIM.FIT заменяются в приложении, auth-экранах, PWA-текстах, manifest, favicon title, HTML metadata и документации пользователя.
- Технические storage keys и Firebase singleton keys не переименовываются, чтобы не потерять локальные данные и не создать две Firebase-инстанции.

## Умный account control

- В мобильном верхнем хедере слева остаётся бренд, справа — баллы и account control размером не меньше 44 px.
- Для гостя control показывает нейтральную иконку профиля и ведёт на `/login`; доступное имя объясняет, что вход включает синхронизацию.
- Для авторизованного пользователя control показывает Google-фото или выбранный 3D-аватар и ведёт в `/settings`.
- Маленький status dot находится на аватаре: лаймовый — данные синхронизированы, фиолетовый пульс — синхронизация идёт, серый — локальный/offline, красный — ошибка.
- Отдельный жёлтый статус подтверждения email на аватаре не используется: подтверждение уже объясняется существующим баннером и настройками.
- Desktop sidebar продолжает показывать расширенную строку профиля, используя ту же модель аватара и статуса.
- Immersive active-workout layout остаётся без account control и отвлекающих баннеров.

## Аватары

- В приложение добавляются десять дружелюбных 3D-портретов людей с разной внешностью, единым светом, ракурсом и нейтральным фоном.
- На регистрации один аватар выбран автоматически; выбор не создаёт дополнительный обязательный шаг.
- Пользователь может открыть компактный picker и выбрать другой аватар до создания email-аккаунта.
- Для Google-входа приоритет имеет фотография Google. В настройках пользователь может переключиться на любой встроенный аватар.
- Выбранный avatar ID хранится в профиле приложения; UI не зависит от абсолютного production URL. Для отображения используется локальная таблица `avatarId -> /avatars/<file>`.
- При сохранении встроенного аватара Firebase `photoURL` не подменяется относительным URL. Google photo остаётся доступным как исходный вариант, а выбранный avatar ID хранится в Firestore profile metadata.

## Данные профиля

- `users/{uid}` расширяется безопасными полями `avatarId`, `displayName`, `googlePhotoURL`, `updatedAt`.
- Firestore Rules разрешают владельцу читать и изменять только собственный профиль; существующая защита вложенных данных сохраняется.
- Для гостя avatar ID хранится локально в settings и не мешает последующей миграции тренировок.
- Повреждённый или неизвестный avatar ID нормализуется к первому встроенному аватару.

## Настройки

- Account card показывает крупный текущий аватар, имя/email и состояние синхронизации.
- Кнопка `Изменить аватар` раскрывает сетку 5×2 на desktop и 4/3 колонки на телефоне без горизонтального overflow.
- Доступен вариант `Фото Google`, только если у пользователя есть Google photo URL.
- Выбор сохраняется сразу с понятным live-status; неуспешная облачная запись не удаляет локальный выбор.

## Auth и домен

- Текущий Email/Password и Google Auth не блокируют реализацию header/avatar UX.
- Переход на passwordless email-link и полностью брендированное письмо выполняется отдельным этапом после регистрации `keepatit.me`, добавления домена в Firebase Authorized domains и настройки DNS/email-провайдера.
- До этого приложение не обещает кастомный sender domain и не скрывает возможную доставку стандартного Firebase-письма в Spam.

## Доступность и responsive

- Touch targets — не меньше 44×44 px; аватар всегда имеет текстовое доступное имя.
- Выбранный аватар отмечается не только цветом, но и checkmark/`aria-pressed`.
- Picker управляется клавиатурой, возвращает фокус на кнопку открытия и сообщает о сохранении через live-region.
- На `390×844` хедер остаётся в одну строку, имя бренда не сталкивается с баллами и аватаром.
- Изображения имеют фиксированные размеры, `object-fit: cover` и не вызывают layout shift.

## Проверки

- Unit: нормализация avatar ID, приоритет Google photo/встроенного аватара, mapping status dot.
- Component: guest control, signed-in control, Google photo, picker, keyboard, сохранение и error state.
- Regression: existing auth, cloud sync, settings, PWA manifest и active workout.
- Visual: `390×844` и `1440×900`, отсутствие overflow, читаемость каждого аватара в 40–44 px.
- Финальный gate: `npm run check`, Firebase rules tests, production build и релевантный E2E smoke.

## Вне текущего этапа

- Покупка домена, DNS, SMTP/Resend и Cloud Functions.
- Юридическая регистрация товарного знака.
- Загрузка пользовательских фотографий и Firebase Storage.
