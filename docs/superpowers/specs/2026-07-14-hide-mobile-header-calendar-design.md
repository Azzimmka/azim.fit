# AZIM.FIT — скрытие календарной кнопки в мобильном заголовке

## Проблема

На ширине до `820 px` кнопка `Запланировать` в `PageHeader` превращается в квадратную иконку календаря без текста. На экранах `Сегодня`, `Календарь тренировок` и `Твой прогресс` она выглядит как посторонний логотип справа от заголовка.

## Решение

- Скрыть `.header-actions .primary-button` внутри существующего breakpoint `@media (max-width: 820px)`.
- Не менять desktop-версию: при ширине больше `820 px` кнопка `Запланировать` остаётся видимой с текстом.
- Не удалять `onAdd` и не менять API `PageHeader`, чтобы desktop-поведение и другие потребители компонента сохранились.
- Не затрагивать кнопки добавления внутри hero-блока, empty-state, календаря и других частей страниц.
- Не менять иконки календаря, которые обозначают настоящий календарь или дату.

## Проверка

- На `390×844` и ширине `820 px` квадратная кнопка календаря справа от заголовка отсутствует.
- На `1440×900` текстовая кнопка `Запланировать` остаётся видимой и рабочей.
- Заголовок занимает освободившееся место без горизонтального переполнения.
- Production build завершается успешно.
