# AZIM.FIT — широкие столбцы прогресса на компьютере

## Цель

Сделать серые вертикальные треки и их зелёное заполнение в недельном графике заметно шире на компьютерах, не меняя сетку из семи дней и компактный мобильный вид.

## Утверждённый дизайн

- Вариант: `B+ Power Blocks`.
- Для viewport шире `820px` трек получает ширину `min(120px, 88%)` и скругление `19px`.
- Зелёное заполнение всегда занимает полную ширину серого трека и получает такое же скругление.
- Сетка, расстояние между днями, подписи, значения, высота графика, hover и selected-состояния сохраняются.
- На viewport до `820px` включительно остаётся текущая ширина до `38px`; мобильная компоновка не меняется.

## Реализация

Изменение ограничивается desktop media query в `src/styles.css`. JSX, данные, события выбора дня и доменная логика не меняются.

## Проверка

- Production viewport `1440×900`: семь столбцов не пересекаются и визуально заметно шире текущих.
- Mobile viewport `390×844`: размеры совпадают с текущей компактной версией, горизонтального overflow нет.
- Состояния hover, selected и зелёное заполнение сохраняют границы нового трека.
- Финальный gate: `npm run check` и целевой Playwright overflow-сценарий.
