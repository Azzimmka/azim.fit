# AZIM.FIT V2

Личный local-first фитнес-трекер на React и Vite. Данные остаются в браузере и доступны без сети после первого online-запуска PWA.

## Возможности

- месячный календарь, пропущенные тренировки и маршруты `/today`, `/plan`, `/progress`, `/settings`, `/workouts/:id`;
- создание, редактирование, перенос, дублирование, удаление с отменой и конечные повторяющиеся серии;
- независимые шаблоны тренировок;
- выполненные подходы, фактический вес, повторы, RPE, плановые и итоговые заметки;
- дневник массы тела, недельная статистика и личные рекорды по весу, объёму и повторениям;
- один восстанавливаемый после перезагрузки таймер отдыха;
- PWA-установка, offline app shell, подтверждаемое обновление и активные session-напоминания;
- корректные русские формы через `Intl.PluralRules('ru-RU')`.

Экспорт, импорт, облачная синхронизация и backend намеренно не входят в V2.

## Локальный запуск

Требуется Node.js `^20.19.0` или `>=22.12.0`.

```bash
npm install
npm run dev
```

Проверки:

```bash
npm run lint
npm test
npm run build
npm run test:e2e
```

Для первого локального E2E-запуска может потребоваться:

```bash
npx playwright install chromium
```

Полная быстрая проверка без E2E: `npm run check`.

## Хранение и миграция

V2 хранится в `localStorage` под ключом `azim-fit-state-v2`. При первом запуске существующий `azim-fit-state-v1` безопасно мигрирует и не удаляется. Новый профиль начинается пустым; тестовые данные доступны по кнопке «Загрузить демо» в настройках.

Напоминания работают, пока вкладка или установленная PWA активна. Без backend браузер не обязан запускать полностью закрытый service worker по расписанию.

## Развёртывание

Production-сборка создаётся командой `npm run build`. Конфигурация `vercel.json` содержит SPA rewrite и cache headers для корневого HTTPS-развёртывания на Vercel.
